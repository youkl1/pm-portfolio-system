# 需求对齐文档 - 产品经理个人作品展示系统

## 项目与任务特性规范
- **项目名称**：产品经理个人作品展示系统（PM Portfolio Hub）
- **任务类型**：全栈开发任务
- **技术栈**：
  - 后端：Java 17+, Spring Boot 3.x, Spring Data JPA, MySQL, Lombok
  - 前端：Vue 3 (Composition API) + Axios + 原生 CSS
- **数据库**：MySQL 8.0+

## 原始需求原文
> **角色**：你是一名资深的全栈架构师，擅长构建轻量级但结构规范的 Java Web 应用。 
> **任务**：为“产品经理个人作品展示系统（PM Portfolio Hub）”编写完整的全栈代码。 
> **输出结构**：请严格按照以下三个部分依次输出。 
> 
> **⚠️ 第一部分：数据库设计 (MySQL)** 
> 1. 提供 `schema.sql` 脚本，包含建表、默认数据初始化。 
> 2. **表结构要求**： 
>    * **`sys_user` (用户表)**： 
>      - 字段：`id` (主键), `username` (账号), `password` (密码), `role` (角色：'admin' 或 'test'), `expire_time` (过期时间), `created_at`。 
>      - 初始化数据：默认插入两个账号，`admin/admin123` (角色admin)，`test/test123` (角色test)，过期时间设为10年后。 
>    * **`pm_project` (作品表)**： 
>      - 字段：`id` (主键), `title` (标题), `description` (描述), `cover_image` (封面图链接), `detail_link` (详情跳转链接), `sort` (排序), `created_at`, `updated_at`。 
>    * 要求：所有字段必须有 **COMMENT 注释**，包含必要索引。 
> 
> **⚠️ 第二部分：后端服务 (Spring Boot)** 
> * **技术栈**：Java 17+, Spring Boot 3.x, Spring Data JPA, MySQL, Lombok。 
> * **核心逻辑**： 
>   1. **权限控制**： 
>      - 登录成功后将用户信息（含 `role`）存入 ThreadLocal。 
>      - 新增/编辑/删除/上传接口，仅允许 `role='admin'` 的用户访问。 
>   2. **接口定义 (RESTful)**： 
>      - `POST /api/auth/login`：登录。 
>      - `GET /api/projects`：获取列表。 
>      - `POST /api/projects`：新增作品。 
>      - `PUT /api/projects/{id}`：编辑作品。 
>      - `DELETE /api/projects/{id}`：删除作品。 
>      - **`POST /api/upload/cover`**：**新增**，上传封面图片接口。 
>        - 参数：`MultipartFile file`。 
>        - 逻辑：将图片保存到服务器本地 `/tmp/uploads` 目录，生成一个可访问的 URL 路径返回（或为了演示方便，直接返回一个带时间戳的 mock URL 字符串）。 
>   3. **配置**：配置 `CorsConfig` 允许跨域，配置文件上传大小限制。 
> 
> **⚠️ 第三部分：前端代码 (Vue 3)** 
> * 技术栈：Vue 3 (Composition API) + Axios + 原生 CSS。 
> * **功能模块**： 
>   1. **登录页**：略。 
>   2. **作品展示页**： 
>      - 顶部导航：admin 显示“新增作品”。 
>      - 作品卡片：admin 显示编辑/删除，test 仅查看。 
>   3. **新增/编辑弹窗 (核心修改)**： 
>      - 表单字段：标题、描述、**封面上传区**、详情链接、排序。 
>      - **封面上传交互**： 
>        - 显示一个“点击上传图片”的区域。 
>        - 点击后触发 `<input type="file">` 选择本地图片（限制 jpg/png）。 
>        - 选中图片后，**先**调用 `POST /api/upload/cover` 接口。 
>        - 上传成功后，将返回的 URL 填入表单 `coverImage` 字段，并在页面上显示图片预览。 
>   4. **要求**：代码简洁，科技风，响应式，关键逻辑加中文注释。 
>  
> 要围绕项目规则进行开发，数据库服务器地址：8.148.235.131端口3306 账号root 密码123456！

## 业务背景与目标
- **业务背景**：产品经理需要一个个人作品展示系统，用于展示自己的项目案例，方便他人查看和了解。
- **业务目标**：
  1. 提供一个简洁美观的作品展示界面
  2. 支持管理员对作品的增删改查操作
  3. 支持作品封面图片上传
  4. 区分不同角色的权限，确保系统安全

## 任务边界确认
### 明确做什么
1. 数据库设计：创建用户表和作品表，初始化默认数据
2. 后端服务：实现用户认证、作品管理、图片上传接口
3. 前端开发：实现登录页、作品展示页、新增/编辑弹窗
4. 权限控制：区分admin和test角色的访问权限

### 明确不做什么
1. 不实现复杂的用户管理功能（如注册、密码重置等）
2. 不实现作品详情页，仅提供跳转链接
3. 不实现图片存储的持久化方案，仅做演示
4. 不实现高级的前端框架（如Vuex、Pinia等状态管理）

## 需求理解说明
1. **用户认证**：
   - 实现简单的登录功能，验证用户名和密码
   - 登录成功后将用户信息存入ThreadLocal
   - 基于角色进行权限控制

2. **作品管理**：
   - 支持作品的增删改查操作
   - 作品包含标题、描述、封面图片、详情链接、排序等字段

3. **图片上传**：
   - 提供封面图片上传接口
   - 支持jpg/png格式图片
   - 上传后返回可访问的URL

4. **前端交互**：
   - 登录页：简单的登录表单
   - 作品展示页：响应式布局，展示作品卡片
   - 新增/编辑弹窗：包含表单字段和图片上传功能
   - 根据角色显示不同的操作按钮
